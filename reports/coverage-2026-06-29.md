## Résumé exécutif

Aucun commit n'a touché `staging` depuis le 06-13 (cf. `reports/audit-dette-2026-06-29.md`, résumé exécutif). La couverture de tests est donc **strictement identique** à celle du rapport du 06-22 — revérifiée indépendamment ci-dessous plutôt que recopiée, pour la 3ᵉ semaine consécutive.

La couverture reste **globalement très solide** : toutes les routes critiques (auth, scouting, conflits, invitations, guest access, API tokens, commentaires joueurs, historique) sont testées. **5 routes restent sans tests**, toutes liées aux features guest access et bannière groupe — inchangé depuis 14 jours.

**Nouveau cette semaine :** la route `POST /api/rounds/[roundId]/matchs` (cf. `audit-dette-2026-06-29.md` §1.2) est techniquement listée comme "✅ Testé" dans le tableau ci-dessous (le test couvre `round-paginated.test.ts`), mais **uniquement pour le `GET`** — le `POST`, qui accepte un body non validé sans aucun contrôle d'autorisation au-delà de l'auth middleware, n'a aucun cas de test dédié. Faute d'usage frontend identifié pour ce `POST`, la recommandation est de sécuriser/supprimer la route avant de lui écrire des tests (voir SEC-03 dans `stories-2026-06-29.md`).
