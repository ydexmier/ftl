## Résumé exécutif

La couverture de tests est **globalement très solide** : toutes les routes critiques (auth, scouting, conflits, invitations, guest access, API tokens, commentaires joueurs, historique) sont testées. Depuis l'audit du 06-08, quatre nouvelles suites de tests ont été ajoutées (`admin-groups.test.ts` étendu avec pin/memo, `api-tokens.test.ts`, `comment-counts.test.ts`, `player-history.test.ts`).

**5 routes restent sans tests** : elles appartiennent toutes aux features guest access et bannière groupe (magic link, liste guests, my-tournaments, admin-groups, member-groups). Ces routes sont simples mais contiennent de la logique de contrôle d'accès qui mérite d'être vérifiée.

## 6. Tests à écrire — Liste priorisée

### Lot 1 🔴 — Logique de contrôle d'accès critique (guest access)

Ces routes implémentent des contrôles d'accès non triviaux pour le flux guest access.

| Route | Cas à couvrir |
|-------|--------------|
| `GET /api/guest/my-tournaments` | 401 sans session ; 200 vide si pas d'accès ; 200 avec tournois ACCEPTED ; filtrage des accès REVOKED |
| `GET /api/groups/[id]/tournaments/[tid]/magic-link` | 401 sans session ; 403 si non-admin ; 200 avec lien actif ; 200 null si aucun lien |
| `POST /api/groups/[id]/tournaments/[tid]/magic-link` | 401 sans session ; 403 si non-admin ; 404 si tournoi pas dans le groupe ; 201 avec nouveau token UUID ; upsert (désactive l'ancien, crée le nouveau) |

### Lot 2 🟠 — Routes de navigation groupe/tournoi

Ces routes de lecture retournent des données affichées dans l'UI mais contiennent de la logique de filtrage.

| Route | Cas à couvrir |
|-------|--------------|
| `GET /api/tournaments/[id]/admin-groups` | 401 sans session ; 200 vide si aucun groupe ; retourne uniquement les groupes où l'user est admin |
| `GET /api/tournaments/[id]/member-groups` | 401 sans session ; 200 vide ; retourne les groupes membres |
| `GET /api/groups/[id]/tournaments/[tid]/guests` | 401 sans session ; 403 si non-admin ; liste des accès externes du tournoi |

| Route | Cas à couvrir |
|-------|--------------|
