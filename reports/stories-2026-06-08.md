## SEC-01 — Rate limiting sur `/api/guest/validate`

**Source :** Rapport audit (sécurité)  
**Priorité :** 🔴 Critique  
**Effort :** XS

### Contexte

`POST /api/guest/validate` est un endpoint public qui crée un compte utilisateur (`isGuest: true`) et une session Argon2id. Il n'a aucun rate limiting par IP, contrairement à tous les autres endpoints d'inscription du projet (`/api/auth/forgot-password`, `/api/invitations/[token]`, `/api/access-requests`). Un attaquant peut spammer des créations de comptes via le même magic link ou des tokens différents.

### Critères d'acceptation

- [ ] `checkRateLimit('guest-validate:${ip}')` est appelé en début de handler
- [ ] La route retourne 429 avec message lisible après dépassement du seuil
- [ ] La clé de rate limit est distincte des autres clés existantes
- [ ] Un test d'intégration vérifie que la 6ème tentative depuis la même IP retourne 429
- [ ] Aucune régression sur le flux normal d'inscription invité

### Dépendances

Aucune.

- [ ] `connectToMongoDB()` et son import sont retirés de `external/.../players/route.ts`
- [ ] Les tests existants (`api-tokens.test.ts`) continuent de passer sans modification

## REFACTO-01 — Centraliser `getIp()` dans `src/lib/auth/`

**Source :** Rapport audit (redondances)  
**Priorité :** 🟡 Mineur  
**Effort :** XS

### Contexte

La fonction d'extraction d'IP depuis `x-forwarded-for` est copiée-collée à l'identique dans 6 fichiers. Toute modification future (support d'IPv6 formaté, en-têtes alternatifs comme `x-real-ip`) devra être répercutée dans 6 endroits.

**Fichiers concernés :**
`app/api/auth/login/route.ts`, `app/api/auth/forgot-password/route.ts`, `app/api/access-requests/route.ts`, `app/api/auth/reset-password/[token]/route.ts`, `app/api/invitations/[token]/route.ts`, `app/api/guest/validate/route.ts`

### Critères d'acceptation

- [ ] `src/lib/auth/getIp.ts` existe avec la fonction `export function getIp(request: NextRequest): string`
- [ ] Les 6 fichiers importent `getIp` depuis ce module et suppriment leur définition locale
- [ ] Les tests d'intégration existants (login, forgot-password, invitations, guest) continuent de passer

### Dépendances

SEC-01 (pour que la correction du rate limiting en bénéficie directement).

## TEST-01 — Tests routes admin `pin` et `memo`

**Source :** Rapport couverture  
**Priorité :** 🟠 Important  
**Effort :** S

### Contexte

Trois routes créées pour les features récentes (épinglage de groupe admin, mémo admin) n'ont aucun test d'intégration. Elles contiennent de la logique d'état (unicité du groupe épinglé, contraintes de longueur du mémo) qui n'est pas couverte.

**Routes :**
- `GET /api/admin/groups/pinned`
- `POST /api/admin/groups/[id]/pin`
- `DELETE /api/admin/groups/[id]/pin`
- `GET /api/admin/users/[id]/memo`
- `PUT /api/admin/users/[id]/memo`

### Critères d'acceptation

- [ ] `GET /api/admin/groups/pinned` retourne null si aucun groupe épinglé
- [ ] `POST /api/admin/groups/[id]/pin` épingle un groupe et retourne `{ isPinned: true }`
- [ ] Épingler un nouveau groupe dépingle l'ancien (un seul groupe épinglé à la fois)
- [ ] `DELETE /api/admin/groups/[id]/pin` désépingle et `pinned` retourne null
- [ ] `GET /api/admin/users/[id]/memo` retourne null si aucun mémo
- [ ] `PUT /api/admin/users/[id]/memo` crée ou met à jour le mémo
- [ ] `PUT` retourne 400 si contenu vide ou > 2000 caractères
- [ ] Routes retournent 403 pour un USER non-admin
- [ ] Tests dans `admin-groups.test.ts` (pin) et un nouveau fichier ou dans `admin-users.test.ts` (memo)

### Dépendances

Aucune.

## TS-01 — Typer les pipelines MongoDB avec `PipelineStage[]`

**Source :** Rapport audit (dette TypeScript)  
**Priorité :** 🟡 Mineur  
**Effort :** S

### Contexte

7 usages de `any` dans les repositories `RoundRepository.ts` et `TournamentPlayersDeckRepository.ts` servent à typer des pipelines d'agrégation MongoDB. Mongoose v7 expose `PipelineStage` (depuis `mongoose`) qui permet de typer correctement les pipelines sans perdre la flexibilité.

**Fichiers :**
- `src/repositories/db/RoundRepository.ts` — lignes 67, 95, 114, 167
- `src/repositories/db/TournamentPlayersDeckRepository.ts` — lignes 142, 163, 224

### Critères d'acceptation

- [ ] `import type { PipelineStage } from 'mongoose'` est ajouté dans les deux fichiers
- [ ] `any[]` remplacé par `PipelineStage[]` pour les tableaux de pipeline
- [ ] `any` sur `rawDecks` (ligne 67 de `RoundRepository.ts`) remplacé par le type approprié depuis le modèle
- [ ] `npm test` passe à 100%
- [ ] Aucun `tsc --noEmit` error sur le projet

### Dépendances

Aucune.
