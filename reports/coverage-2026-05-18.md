## Liste priorisée des tests à écrire

### Lot 1 — 🟠 Important

#### 1. `src/__tests__/hooks/mergeDeep.test.ts` — Tests unitaires

**Fichier cible :** `src/lib/mergeDeep.ts`

Tester :
- `mergeDeep({ a: 1, b: { c: 2 } }, { b: { d: 3 } })` → `{ a: 1, b: { c: 2, d: 3 } }` (merge récursif)
- `mergeDeep({ a: [1, 2] }, { a: [3] })` → `{ a: [3] }` (les tableaux ne sont pas mergés, ils sont remplacés)
- `mergeDeep({}, { a: 1 })` → `{}` (source key inexistante dans target → ignorée)
- `mergeArrayById([{ id: 1, name: 'a' }], [{ id: 1, name: 'b' }, { id: 2, name: 'c' }])` → `[{ id: 1, name: 'b' }, { id: 2, name: 'c' }]`
- `mergeArrayById([], [{ id: 1, name: 'a' }])` → `[{ id: 1, name: 'a' }]`
- `mergeArrayById([{ id: 1, name: 'a' }], [])` → `[{ id: 1, name: 'a' }]`

> `mergeDeep` et `mergeArrayById` sont utilisées dans `RoundRepository.mergeAndSave` et `TournamentRepository.mergeAndSave`. Un comportement silencieusement incorrect ici peut corrompre des données de tournoi.

#### 2. `src/__tests__/middleware.test.ts` — Tests unitaires (logique pure)

**Fichier cible :** `middleware.ts`

Approche : importer et tester directement la logique de décision, sans le runtime Next.js Edge.

Tester :
- Route publique (`/login`, `/api/auth/login`) → pas de vérification de cookie
- Route protégée sans cookie → redirect `/login`
- Route protégée avec cookie HMAC valide → passage (200)
- Route protégée avec cookie HMAC invalide → redirect `/login` (pas d'erreur 500)
- Route `/admin/` avec rôle USER → redirect `/`
- Route `/admin/` avec rôle ADMIN → passage

> Le middleware est la première ligne de défense. Un bug ici affecte toutes les routes protégées.

## Observation sécurité (hors scope tests)

`POST /api/tournaments` (`app/api/tournaments/route.ts`) effectue un `upsert` direct de données tournoi sans aucune vérification d'authentification. N'importe quel client peut écraser les données d'un tournoi. Le test `tournament-assign.test.ts` confirme ce comportement (appel sans cookie → 200). Si cette route est destinée à être interne (seuls les admins ou le système peuvent upsert des tournois), elle doit être protégée par `getAuthSession` + `hasRole('ADMIN')`.
